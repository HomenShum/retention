# retention.sh: Open vs Closed Strategy

*Core principle: Open the habit layer. Own the compounding intelligence layer.*

---

## Decision

We do not fully open source the whole product. We do not fully close everything.

Best path: open the adoption layer, keep the compounding intelligence layer proprietary.

---

## What We Open Source

Open source the parts that help people try TA quickly and understand the product value.

### Open Pieces

| Component | Purpose |
|-----------|---------|
| MCP tools / local starter kit | Low-friction local usage |
| Benchmark app templates | Reproducible demos |
| Sample workflows and runbooks | Show how TA works |
| TCWP schema spec | Industry trust + standardization |
| Replay / trajectory artifact schema | Interoperability |
| Failure bundle schema | Community adoption |
| Basic CLI / local runner | Entry point |
| Example benchmark cases | Proof methodology |

### Why Open:
- Lowers friction for friends and community
- Makes adoption feel safe and transparent
- Creates trust because users can inspect the workflow
- Gives distribution through GitHub, MCP registries, docs, community sharing
- Lets people integrate TA into Claude Code / OpenClaw / local workflows fast

---

## What We Keep Proprietary

Keep the parts that compound over time and become the true moat.

### Closed Pieces

| Component | Why Proprietary |
|-----------|----------------|
| Web dashboard / operating workspace | Core product surface |
| Trajectory memory graph | Compounds with usage |
| Replay optimization + checkpoint validation | Core differentiation |
| Rerun targeting logic | Intelligent replay selection |
| Trajectory comparison engine | Shortcut discovery |
| Shortcut recommendation + audit engine | Workflow compression moat |
| Longitudinal rollups (day/week/month/quarter/year) | Time-series intelligence |
| Account-level local-to-cloud profile sync | Platform stickiness |
| Benchmark comparison dashboards | Sales-ready artifacts |
| Shared team memory + enterprise workflow layer | Enterprise upsell |
| Workflow compression / downstream intelligence | Deepest moat |

### Why Closed:
- This is where long-term value accumulates
- Competitors cannot clone this from seeing open starter tools
- This is what we can price on later
- Compounding data advantage over time

---

## Product Split

### TA Starter / TA MCP Kit (Open)
- Fast setup
- Local usage
- Sample workflows
- Reproducible benchmark
- Schemas and artifacts
- Enough value to get people hooked

### retention.sh Cloud (Closed)
- Dashboard
- Memory + trajectory registry
- Savings visibility
- Before/after comparisons
- Rerun intelligence
- Portfolio and team views
- Profile sync
- Enterprise extensions later

---

## Why This Is the Right Tradeoff

| Strategy | Pro | Con |
|----------|-----|-----|
| Fully open source | Trust, community, distribution | Give away company before building gravity |
| Fully closed | Preserve ownership | Lose trust, community spread, adoption |
| **Open-core (our strategy)** | **Trust + ownership** | **Requires clear boundary discipline** |

We are not doing vague "open-core." We are doing:
**Open workflow adoption, closed compounding memory + operating layer.**
